**名师议课堂**是一个面向一线语文教师的 **AI 圆桌备课系统**。它把王崧舟、薛法根、窦桂梅等语文名师的教学理念，蒸馏成可对话的 AI 角色，模拟一场围绕你这节课的名师研讨会。

- **给谁**：备课没把握、想要多视角教学建议的中小学语文老师；师范生、教研员、备课组。
- **不是什么**：不是帮你"一键生成教案"的偷懒工具。它的价值在于**让你看见不同流派如何思考同一节课**，在思想碰撞里想清楚自己要上一堂什么样的课。
- **名师画像**均基于公开的课堂实录、著作与讲座提炼，仅供教学研究与备课启发使用。

## 它做了什么

把「一个人对着课文枯坐」变成「七位名师围着你这节课开研讨会」。

```
填课文信息  →  选名师上桌  →  三轮圆桌研讨（你可随时插话）  →  生成教案草稿  →  导出 MD / PDF / Word
```

你拿到的不是一份冷冰冰的模板教案，而是**一整场有观点、有交锋、有共识的备课研讨**——以及它沉淀出的结构化教案。

| 产出 | 给谁 | 解决什么 |
|------|------|---------|
| **名师研讨全记录** | 给你看 | 七种流派如何解读同一篇课文，思路、分歧、理由一目了然 |
| **结构化教案草稿** | 给你用 | 研讨收束成一份可直接改用的教案初稿 |
| **可导出文档** | 给你存 | Markdown / PDF / Word 三种格式，带走整场研讨与教案 |

## 名师阵容

七位语文名师，每位都按其真实理念、方法论、语言风格"像素级"还原——连标志性口头禅、对其他流派的态度都在。研讨时他们会从各自立场发言，也会**针锋相对地交锋**。

| 名师 | 称号 | 核心主张 | 在桌上扮演什么视角 |
|------|------|---------|------------------|
| **王崧舟** | 诗意语文创始人 | 诗意语文 · 入乎其内出乎其外 | 意象、节奏、情感、语言审美 |
| **薛法根** | 组块教学创始人 | 组块教学 · 清简 · 一课一得 | 文章逻辑结构、语文要素 |
| **窦桂梅** | 主题教学创始人 / 清华附小校长 | 主题教学 · 大语文 | 跨文本联结、拓展视野 |
| **管建刚** | 作文教学革命倡导者 | 习作教学 · 真实写作 | 读写结合，追问"怎么迁移到写作" |
| **于永正** | 五重教学创始人 | 五重教学 · 朗读为本 | 朗读设计，质疑"学生真读进去了吗" |
| **孙双金** | 情智教育创始人 | 情智语文 | 情感共鸣，问"孩子的情感被触动了吗" |
| **蒋军晶** | 群文阅读代表人物 | 群文阅读 | 阅读策略与趣味，倡导"有意思的语文" |

> 比如让王崧舟和薛法根同桌——一个说"情感体验才是大厦本身"，一个回"伤其十指不如断其一指，这节课最核心的一得是什么"。流派的真实分歧，就这样摆到你眼前。

## 三轮研讨

一场完整的圆桌，分三轮层层推进。**每一轮结束后，你都可以插话**——追问、补充、提出你的顾虑，名师们会带着你的意见进入下一轮。

| 轮次 | 主题 | 发生什么 |
|------|------|---------|
| **第一轮** | 🗣️ 各抒己见 | 每位名师从自己的流派出发，独立给出对这节课的建议 |
| **第二轮** | ⚔️ 观点交锋 | 名师们互相看到彼此的发言，开始碰撞、反驳、补强 |
| **第三轮** | 🤝 达成共识 | 在交锋的基础上收束，给出清晰、可落地的最终建议 |

```
第一轮（各抒己见）  →〔你插话〕→  第二轮（观点交锋）  →〔你插话〕→  第三轮（达成共识）  →  生成教案
```

> 你不是旁观者，而是这场研讨的主持人。随时可以留空快速推进，也可以深度参与把研讨"问"向你想要的方向。

## 自定义名师

七位之外，还想请别人上桌？**贴一段资料，AI 帮你"造"一位名师。**

把某位教育家的理念、课堂实录或著作片段粘进去，系统会自动提炼出他的核心理念、教学方法论、语言风格与标志性句式，生成一个可参与圆桌的 AI 角色。

```
粘贴专家资料  →  AI 提炼画像（理念 / 方法 / 风格 / 互动方式）  →  新名师加入圆桌
```

> 自定义名师保存在你浏览器本地（localStorage），不上传服务器。

## 30 秒上手

**前置要求：** Node.js

```bash
# 1. 克隆
git clone https://github.com/hututu-ai/AI-MASTER-TEACHER-ROUNDTABLE.git
cd AI-MASTER-TEACHER-ROUNDTABLE

# 2. 装依赖
npm install

# 3. 启动
npm run dev
```

浏览器打开终端提示的本地地址（默认 **http://localhost:3000**），在应用内「设置」里填一次 API（见下），即可开始第一场研讨。

---

## API 配置

本应用调用 **OpenAI 兼容接口**来驱动名师对话，用你自己的 Key，灵活又省钱。在应用内 **设置（Settings）** 面板里填写：

| 字段 | 填什么 | 举例 |
|------|--------|------|
| **API 地址** | 供应商接口地址 | `https://api.deepseek.com` |
| **API Key** | 你的密钥 | `sk-xxxxxx` |
| **模型名称** | 要用的模型（默认 `deepseek-chat`） | `deepseek-chat` |

兼容 DeepSeek、Kimi / Moonshot、通义千问、智谱、OpenAI 等主流供应商。配置保存在浏览器本地。

> 💡 名师研讨是多角色、多轮长文本任务，DeepSeek 等国产模型性价比很高，一场研讨成本很低。
